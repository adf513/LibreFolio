# F-098 — Progressive Web App (PWA)

## What it does
Enables users to install LibreFolio as a native-like app on mobile (iOS/Android) and desktop. Provides an offline fallback page when the network is unavailable. Removes browser chrome in standalone mode for full-screen experience.

## Layer breakdown
**Frontend**: Service Worker (`service-worker.js`), manifest (`manifest.json`), offline page (`offline.html`), install button in HelpMenu, dynamic `theme-color` meta tag, iOS safe-area padding, mobile CSS optimizations (no-zoom, no-overscroll, no-tap-highlight).

**Infrastructure** (dev.py): Icon generation via PIL (192×192, 512×512 with 12% white padding), `stamp_service_worker()` for auto-versioning via MD5 hash.

## Dependencies
- [[F-007]] Theme System — PWA syncs `theme-color` meta tag with dark/light mode

## Implementation notes
- **No app caching**: Service Worker intentionally does NOT cache app assets. For a financial app, data freshness is critical — stale cache could show wrong portfolio values. Only the offline fallback page is cached.
- **HTTP LAN works**: `manifest.json` + standalone mode work over plain HTTP. Only Chrome's `beforeinstallprompt` auto-banner requires HTTPS. Documented Tailscale workaround for LAN HTTPS.
- **iOS limitations**: iOS never shows auto-install banner. HelpMenu detects iOS and shows "Share → Add to Home Screen" manual instructions.
- **Android**: Detects Android to show "HTTPS required for auto-install" message with Tailscale hint.
- **Flag emoji fix**: `.emoji-flag` font-family reordered — Apple Color Emoji first (iOS native flags), Noto Color Emoji fallback (Windows/Linux).
- **Safe area**: `env(safe-area-inset-top)` padding for Sidebar, Header, login toolbar in iOS standalone mode. Zero impact on desktop/Android (env() resolves to 0px).
- **Auto-versioning**: `dev.py stamp_service_worker()` injects content hash → browser always picks up SW updates.

## Dead ends / Ideas explored
- **Full app caching via SW**: Rejected — financial data must always be fresh; cache invalidation complexity not worth the offline benefit for a portfolio tracker.
- **Workbox integration**: Rejected — overkill for offline-fallback-only strategy; hand-written SW is 30 lines.
