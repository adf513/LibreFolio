# F-042 — FX Pair Comparison Signal

## What it does
Overlay signal that plots a configured FX pair's rate history on top of any asset or FX chart for comparison. Data is pre-fetched by the parent component from the `fxStoreRegistry` `TimeSeriesStore` and injected via `params._resolvedData`. In percentage mode, the FX pair is normalized to its own starting value (not the base chart's), enabling relative performance comparison despite different scales.

## Layer breakdown
**Frontend**: `frontend/src/lib/charts/signals/FxPairSignal.ts`, `frontend/src/lib/stores/fxStoreRegistry.ts` (data source)

## Dependencies
- [[F-023]] FX Chart & Signal Overlay — chart host
- [[F-017]] FX Rate Sync — data must be synced before comparison is possible

## Enables
- (cross-chart analysis — no downstream)

## Implementation notes
- `signalType`: `fx-pair`, `category`: `comparison`, `icon`: 💱
- No `maxInstances` limit — multiple FX pairs can be overlaid
- **Parameters**:
  - `pairSlug` (select, `dynamicOptionsKey: 'configuredFxPairs'`) — runtime-resolved dropdown
- `params._resolvedData` injected by `PriceChartFull` before `computePoints()` — `_` prefix excludes from serialization
- Inversion support: `params._inverted = true` → shows 1/rate
- Percentage normalization: each series normalized to its own first value
