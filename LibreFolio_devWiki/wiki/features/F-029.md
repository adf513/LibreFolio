# F-029 — Asset Metadata Refresh

## What it does
Fetches up-to-date metadata (full name, sector, country, ISIN, asset type) from the provider for an asset and shows a diff between the stored values and the provider's current data. Users can choose to apply the update or keep existing values. Useful for correcting auto-filled data or refreshing after a corporate action.

## Layer breakdown
**Backend**: `backend/app/api/v1/assets.py` (POST `/assets/metadata/refresh` bulk), `backend/app/services/asset_source.py` (`AssetSourceManager.fetch_asset_metadata()`), `backend/app/schemas/assets.py` (`FAAssetMetadataResponse`, `OldNew` diff fields)

**Frontend**: `frontend/src/lib/components/assets/ProviderComparisonModal.svelte` (323 lines), refresh button in asset detail

## Dependencies
- [[F-026]] Provider Assignment — metadata fetched from assigned provider
- [[F-025]] Asset Provider Registry — `fetch_asset_metadata()` optional method on providers

## Enables
- [[F-036]] Provider Comparison Modal — the modal shows old vs new values

## Implementation notes
- `fetch_asset_metadata()` is optional on providers — only yfinance and JustETF implement it
- Metadata cache TTL: 30 min
- `OldNew[T]` schema in `common.py`: `{old: T, new: T}` — used for each changed field in the diff
- `FABulkMetadataRefreshResponse` wraps per-asset results
