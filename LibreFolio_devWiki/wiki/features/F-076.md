# F-076 — Log Level Policy & TRACE Level

## What it does
Formalizes the backend logging policy with a custom `TRACE=5` level (below DEBUG), structlog extension for structured JSON logging, and a policy document in `logging_config.py` that defines what goes at each level.

## Implementation notes
- Add `TRACE = 5` custom level to Python logging
- Extend structlog configuration in `logging_config.py`
- Define level policy: TRACE=provider internals, DEBUG=service calls, INFO=API requests, WARNING=recoverable errors, ERROR=exceptions
- Medium priority — improves debuggability for provider issues
