# F-090 — AssetEvent → Transaction Link (Enrichment)

## What it does
An enrichment layer on top of [[F-051]] (Transaction ↔ AssetEvent Link). While F-051 adds the FK column, F-090 adds the full UX and business logic: when a provider change is detected (e.g., Yahoo Finance dividend event vs existing INTEREST transaction), show a modal to link or unlink the transaction from the event. Includes CASCADE protection (409 Conflict on event delete if linked transactions exist).

## Note on relation to F-051
F-051 provides the FK infrastructure. F-090 is the enrichment workflow on top of it: provider-change modal, 409 conflict guard, and retroactive linking UI. They are kept separate because F-051 may be implemented without the full enrichment UX.

## Implementation notes
- `asset_event_id FK` on `Transaction` (from F-051)
- CASCADE protection: `DELETE /asset-events/{id}` returns 409 if linked transactions exist
- Provider change modal: shows old event → new event diff, proposes re-link
- `GET /transactions?asset_event_id={id}` for reverse lookup
- HIGH priority for Phase 7
