# F-027 — Asset Data Sync

## What it does
Fetches historical OHLCV price data and current prices for assets from their assigned providers. Sync is triggered manually via the asset list ("Sync All") or per-asset via the detail page. Results are stored in `price_history` (one record per day) and `asset_event` (dividends, splits, etc.). Sync status and coverage dates are reported per asset.

## Layer breakdown
**Backend**: `backend/app/api/v1/assets.py` (POST `/assets/refresh` bulk, GET `/assets/{id}/current-price`), `backend/app/services/asset_source.py` (`AssetSourceManager.sync_asset_prices()`, `fetch_current_price()`), `backend/app/db/models.py` (`PriceHistory`, `AssetEvent`)

**Frontend**: `frontend/src/lib/components/assets/AssetSyncModal.svelte`, sync buttons in list and detail pages

## Dependencies
- [[F-026]] Provider Assignment — sync requires at least one provider assigned
- [[F-025]] Asset Provider Registry — providers called during sync

## Enables
- [[F-030]] Price History — sync populates the OHLCV table
- [[F-033]] Asset Detail Page — chart shows synced data

## Implementation notes
- `FARefreshItem` / `FABulkRefreshResponse` schemas track per-asset sync results
- Smart range fetching: sync only fetches missing date ranges (gap detection in `TimeSeriesStore`)
- Current price TTL cache: 2 min (prevents frontend polling overload)
- History cache: 15 min per asset per date range
- Thread isolation: each provider call in its own thread (`_run_provider_in_thread`)
