# F-086 — Client-side Image Preview Cache (LazyImage)

## What it does
A `LazyImage` component (migrated from Svelte 4 to Svelte 5) backed by a `Map<fileId, {objectUrl, maxWidth}>` cache. Avoids re-fetching images on view toggle (card↔table). Higher-resolution cached images reused for lower-resolution requests via CSS object-fit.

## Implementation notes
- `imagePreviewCache.ts`: plain `.ts` module (no Svelte reactivity needed), size-based reuse
- **No ref counting** — objectUrls held for page lifetime (~2MB/100 images). `clearImagePreviewCache()` on logout
- LazyImage dual mode: `src` prop (backward compat) or `fileId`+`previewUrl` (cache mode)
- `FileGrid.svelte` also migrated Svelte 4→5 in same plan
- `AbortController` in `$effect` cleanup for proper fetch cancellation
- Completed: 2026-06-01
