# F-001 — User Authentication & Sessions

## What it does
Provides login, logout, and session management for LibreFolio. Users authenticate with username/email and password; the server issues a JWT stored in an HttpOnly cookie named `session`. The cookie is sent automatically on every request, enabling stateless multi-worker auth without a session store.

First-registered user is automatically promoted to admin. Inactive users are blocked at the `get_current_user` dependency level with HTTP 401.

## Layer breakdown
**Backend**: `backend/app/api/v1/auth.py` (endpoints: POST `/auth/login`, POST `/auth/logout`, GET `/auth/me`, POST `/auth/register`, POST `/auth/change-password`, PATCH `/auth/profile`), `backend/app/services/auth_service.py` (JWT creation/decode, bcrypt hashing), `backend/app/services/user_service.py`

**Frontend**: `frontend/src/lib/stores/auth.ts`, login/register pages under `frontend/src/routes/`

## Implementation notes
- Cookie name: `session`, HttpOnly, SameSite=lax, max-age configurable via `GlobalSetting` `session_ttl_hours`
- JWT payload carries only `user_id` (UUID); user record fetched from DB on every request
- `get_current_user` FastAPI dependency used across all protected routes
- `get_admin_user` dependency raises 403 if role is not `admin`
- Password hashing: bcrypt via `passlib`
