# F-012 — BRIM Framework (Broker Report Import)

## What it does
A pipeline for importing broker transaction reports. Users upload a file, the system detects or selects the appropriate plugin, parses the file, reviews the extracted transactions in a staging area, resolves unknown assets, and commits to the database.

## Layer breakdown
**Backend**: `backend/app/services/brim_provider.py` (abstract base), `brim_providers/` (11 plugins)
**Frontend**: `frontend/src/lib/components/brokers/` (BrokerImportFilesModal, AssetMatchingWizard)

## Flow
```
Upload file (F-011) → Detect plugin (F-013) → Parse → Review staging (F-049) → Asset matching → Commit (F-046)
```

## Implementation notes

### Fake Asset ID Flow (key design)
BRIM plugins output transactions with **negative fake asset IDs** during parsing. The frontend then maps these to real assets via the asset matching UI. This two-phase approach allows import without requiring pre-existing assets.

### Parse result structure
`parse_file()` → returns `(transactions, warnings, extracted_assets)`
- `extracted_assets`: hints about assets found in the file (ISIN, ticker, name) for the matching UI
- `warnings`: non-fatal issues (missing prices, unknown asset types)

### Plugin version
Each plugin has a `plugin_version` field for cache invalidation — if a plugin is updated, files previously parsed with the old version should be re-parsed.
