# F-086 — Client-side Image Preview Cache (LazyImage)

## What it does
A `LazyImage` component backed by a `Map<fileId, objectUrl>` client-side cache. Avoids re-fetching the same image on view toggle (card/table switch). Object URLs are created once and reused for the session lifetime.

## Implementation notes
- `Map<fileId, objectUrl>` in a Svelte store or module-level variable
- Object URLs revoked on component unmount or session end
- Avoids duplicate network requests when toggling card↔table view
