# F-030 — Asset Price History OHLCV

## What it does
Stores and serves historical OHLCV (Open/High/Low/Close/Volume) price data for assets. One record per asset per day. Supports bulk upsert (from provider sync), bulk delete, range queries, and manual edits via the data editor in the asset detail page.

## Layer breakdown
**Backend**: `backend/app/api/v1/assets.py` (POST `/assets/prices/upsert` bulk, DELETE `/assets/prices` bulk, POST `/assets/prices/query`), `backend/app/db/models.py` (`PriceHistory`: `asset_id`, `date`, `open`, `high`, `low`, `close`, `volume` — all `Numeric(18,6)`)

**Frontend**: `frontend/src/lib/components/assets/AssetDataEditorSection.svelte` (493 lines) — inline price/event editor

## Dependencies
- [[F-027]] Asset Data Sync — sync populates this table

## Enables
- [[F-033]] Asset Detail Page — chart reads from price history

## Implementation notes
- Daily-point policy: one record per (asset_id, date) — UNIQUE constraint enforced
- `Numeric(18, 6)` for all price columns — consistent precision
- `FAPriceQueryItem` / `FAPriceQueryResponse` — range query with backward-fill for gaps
- `BackwardFillInfo` in response: signals when a gap was filled with the nearest prior price
- Manual edits via `EditBuffer` pattern in `AssetDataEditorSection`
