# F-051 — Transaction AssetEvent Link

## What it does
A foreign key relationship between `Transaction` and `AssetEvent` that enables DIVIDEND/INTEREST transactions to reference the global asset event they derive from. This bidirectional link is the prerequisite for a future "smart assistant" that auto-links historical transactions to their corresponding events retroactively.

## Layer breakdown
**Backend**: `backend/app/db/models.py` — `Transaction.related_asset_event_id` FK to `AssetEvent.id`, `backend/app/services/transaction_service.py` — event suggestion endpoint, `backend/app/alembic/versions/001_initial.py` — migration

## Dependencies
- [[F-046]] Transaction Model — the FK lives on Transaction
- [[F-031]] Asset Events — the target of the FK

## Enables
- (future smart assistant for retroactive event matching — not yet planned)

## Implementation notes
- Phase 7 Part 1: `related_asset_event_id` column added to `Transaction`
- `POST /assets/events/query` used during staging to find matching events within date tolerance
- Tolerance slider (0–7 days) in Staging Modal: matches BRIM DIVIDEND transactions to `AssetEvent.DIVIDEND` by date proximity
- Link is optional (nullable FK) — manually entered transactions don't require an event link
