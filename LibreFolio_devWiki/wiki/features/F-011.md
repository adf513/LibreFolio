# F-011 — File Management upload/list/delete

## What it does
General-purpose file upload, listing, and deletion for broker-scoped files. Files are stored on the server filesystem under `backend/data/{env}/custom-uploads/`. The API validates MIME type to prevent executable uploads and enforces a configurable size limit. Files are referenced by a UUID `file_id`. The Files page shows all uploaded files with metadata (name, size, MIME type, upload date).

## Layer breakdown
**Backend**: `backend/app/api/v1/uploads.py` (POST `/uploads`, GET `/uploads`, GET `/uploads/{file_id}`, DELETE `/uploads/{file_id}`, GET `/uploads/{file_id}/preview`), `backend/app/services/static_uploads.py` (save/list/delete/preview), `backend/app/config.py` (data dir paths)

**Frontend**: `frontend/src/routes/(app)/files/` — file management page

## Dependencies
- [[F-009]] Broker CRUD — files are scoped to a broker context

## Enables
- [[F-012]] BRIM Framework — BRIM uses uploaded files as import source
- [[F-050]] File Preview System — inline preview for uploaded files

## Implementation notes
- Storage paths: `backend/data/prod/custom-uploads/` (prod), `backend/data/test/custom-uploads/` (test)
- MIME validation via `UploadSecurityError` in `static_uploads.py`
- `PREVIEW_CACHE_MAX_MB` env var (default 50MB) — in-memory LRU cache for image thumbnails per worker
- `PreviewCache` class: TTL=1h, size-limited; per-process (each uvicorn worker has its own)
- Global settings `max_upload_mb` enforced server-side
