# F-037 — Signal Library Framework

## What it does
An extensible framework for computing and rendering technical analysis signals (indicators, comparisons, benchmarks, tools) on top of price charts. All signals extend a common abstract base class and are auto-discovered via a registry.

## Layer breakdown
**Frontend only**: `frontend/src/lib/charts/signals/`

Key files:
- `ChartSignal.ts` — abstract base class (contract all signals must implement)
- `registry.ts` — `SIGNAL_REGISTRY` map, `getRegisteredSignalTypes()` for UI auto-discovery

## Enables
- [[F-038]] EMA, [[F-039]] RSI, [[F-040]] MACD, [[F-041]] Bollinger Bands
- [[F-042]] FX Pair Comparison, [[F-043]] Asset Comparison
- [[F-044]] Linear/Compound/Sine Benchmarks
- [[F-045]] Measure Tool

## Implementation notes
- All signals compute in **O(N) iteratively** — no full-array passes
- `paramDescriptors` (static field) drives the UI: `ChartSettingsModal` renders param controls dynamically
- `dynamicOptionsKey` in params enables runtime-resolved dropdowns (e.g. available FX pairs)
- MACD is composite — uses `renderMulti()` for 3 series (MACD line + signal + histogram)
- Adding a new signal = 1 file + 1 registry entry — UI auto-discovers via `getRegisteredSignalTypes()`
- Both FX detail page and Asset detail page use the same Signal Library
