# F-031 — Asset Events DIVIDEND/SPLIT/INTEREST

## What it does
Stores and serves discrete asset events: DIVIDEND, SPLIT, INTEREST, PRICE_ADJUSTMENT, MATURITY_SETTLEMENT. Events are emitted by providers during sync (e.g., yfinance returns dividend/split dates, scheduled_investment generates interest milestones) and can also be manually entered. Events are shown on charts as markers and in the data editor timeline.

## Layer breakdown
**Backend**: `backend/app/api/v1/assets.py` (POST `/assets/events/upsert` bulk, DELETE `/assets/events` bulk, POST `/assets/events/query`), `backend/app/db/models.py` (`AssetEvent`: `asset_id`, `event_type`, `event_date`, `details` JSON)

**Frontend**: `frontend/src/lib/components/assets/AssetDataEditorSection.svelte` — events tab, `frontend/src/lib/components/assets/EventDataImportModal.svelte`

## Dependencies

## Implementation notes
- `AssetEvent.event_type` enum: DIVIDEND, INTEREST, PRICE_ADJUSTMENT, SPLIT, MATURITY_SETTLEMENT
- `details` JSON stores event-specific data (amount, ratio, etc.) — schema varies by event type
- `FAAssetEventPoint` schema used in provider results and API responses
- Events integrated in Round 12 (Phase 6 Step 3): yahoo_finance, justetf, scheduled_investment all emit events
