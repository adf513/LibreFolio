# F-059 — Provider Registry Pattern

## What it does
A decorator-based auto-discovery system for all providers (FX, Asset, BRIM). Any class decorated with `@register_provider(SomeRegistry)` is automatically registered at import time. No manual registration or configuration files needed.

## Layer breakdown
**Backend only**: `backend/app/services/provider_registry.py`

Used by three registries:
- `FXProviderRegistry` — for [[F-015]] central bank FX providers
- `AssetProviderRegistry` — for [[F-025]] asset data providers
- `BRIMProviderRegistry` — for [[F-012]] broker import plugins

## Enables
- [[F-015]] FX Provider Registry
- [[F-025]] Asset Provider Registry
- [[F-012]] BRIM Framework

## Implementation notes
- Providers are discovered by importing the module — each provider file's `@register_provider` decorator fires at import
- `params_schema` property on providers drives dynamic forms in the frontend ([[F-026]])
- `dynamicOptionsKey` in params enables runtime-resolved dropdowns (e.g. configured FX pairs in signal options)
- Adding a new provider = create one file + decorate — zero other changes needed
