## §1 — CRITICAL: `list_assets` 500 error — ProviderInputType vs IdentifierType

### Problema

L'endpoint `GET /assets/query` ritorna 500 quando un asset ha `identifier_type = AUTO_GENERATED` nel provider assignment.

```
Error listing assets: 1 validation error for FAinfoResponse
identifier_type
  Input should be 'ISIN', 'TICKER', 'CUSIP', 'SEDOL', 'FIGI', 'UUID' or 'OTHER'
  [type=enum, input_value=<ProviderInputType.AUTO_GENERATED: 'AUTO_GENERATED'>]
```

### Root cause

In `asset_source.py` L2065:
```python
identifier_type=provider_identifier_type,
```
Passa il raw `ProviderInputType` dalla JOIN senza conversione.
`FAinfoResponse.identifier_type` è tipizzato `Optional[IdentifierType]`, che non include `AUTO_GENERATED`.

### Soluzione

Usare `AssetSourceProvider.map_input_type_to_identifier_type()` (L476) per convertire prima di costruire `FAinfoResponse`.
La stessa funzione è già usata correttamente in L1204 e L1641.

```python
# Prima:
identifier_type=provider_identifier_type,

# Dopo:
identifier_type=(
    AssetSourceProvider.map_input_type_to_identifier_type(provider_identifier_type)
    if provider_identifier_type else None
),
```

**File**: `backend/app/services/asset_source.py` L2065

## §2 — BUG: Cambio provider non pulisce params precedenti

### Problema

Utente configura CSS Scraper (URL + selettore), poi cambia a scheduled_investment. Al salvataggio, il payload contiene l'identifier del CSS e i params mescolati.

Payload inviato:
```json
{"identifier":"https://www.borsaitaliana.it/...", "identifier_type":"AUTO_GENERATED",
 "provider_params":{"schedule":[...], "late_interest":null}}
```

### Root cause

In `ProviderAssignmentSection.svelte` L230:
```javascript
function handleProviderChange(code: string) {
    providerCode = code;
    paramsValues = {};     // ← pulisce solo lo state locale
    testStatus = 'not_tested';
    testResults = [];
    emitChange();
}
```
Non resetta `providerParams` (bindable verso AssetModal) né `identifier`. Il vecchio identifier e i vecchi params restano nel parent.

### Soluzione

```javascript
function handleProviderChange(code: string) {
    providerCode = code;
    paramsValues = {};
    providerParams = null;   // ← NEW: pulisce binding verso parent
    identifier = '';          // ← NEW: pulisce identifier
    testStatus = 'not_tested';
    testResults = [];
    emitChange();
}
```

**File**: `frontend/src/lib/components/assets/ProviderAssignmentSection.svelte` L230

## §3 — BUG: DateRangePicker si chiude scrollando dropdown mese/anno

### Problema

Dentro il calendario del DateRangePicker, aprendo il dropdown SimpleSelect per mese o anno e scrollando le opzioni, il popover intero si chiude.

### Root cause

In `DateRangePicker.svelte` L390:
```javascript
const handleScroll = () => closeCalendar();
window.addEventListener('scroll', handleScroll, true);
```
`capture: true` intercetta TUTTI gli scroll events, inclusi quelli del dropdown SimpleSelect interno.

### Soluzione

Modificare il handler per ignorare scroll originati dentro il popover:
```javascript
const handleScroll = (e: Event) => {
    if ((e.target as Element)?.closest?.('.drp-popover')) return;
    closeCalendar();
};
```

Stesso fix in `SingleDatePicker.svelte` L116 che ha lo stesso pattern.

**File**: `DateRangePicker.svelte` L390, `SingleDatePicker.svelte` L116

## §4 — Test history "not supported" → warning giallo

### Problema

Quando CSS Scraper viene testato, la riga History appare in ROSSO con errore ❌.
È un comportamento atteso (il provider non supporta history), dovrebbe essere giallo ⚠️.

### Root cause

`TestResult` ha solo `success: boolean`. Non distingue "errore" da "non supportato".

### Soluzione

1. Estendere `TestResult` con `status?: 'success' | 'error' | 'warning'`
2. Dopo aver costruito gli items, se `success === false` e `detail` contiene `NOT_IMPLEMENTED` o `not supported`: impostare `status = 'warning'`
3. Nel template: `status === 'warning'` → `AlertTriangle` in `text-amber-500` invece di `AlertCircle` in `text-red-500`
4. Per `testStatus`: se tutti i `success` sono true O tutti i `!success` sono `warning` → `passed`

**Import**: aggiungere `AlertTriangle` da lucide-svelte

**File**: `ProviderAssignmentSection.svelte`

## §5 — Tooltip errore: summary inline ≠ dettaglio tooltip

### Problema

In caso di errore, sia l'inline che il tooltip mostrano lo stesso `result.detail` raw.
L'inline dovrebbe mostrare un messaggio corto, il tooltip il dettaglio completo per debug.

### Soluzione

1. Aggiungere `summary?: string` a `TestResult`
2. Funzione `summarizeError(detail: string): string`:
   - Contiene `NOT_IMPLEMENTED` → "Not supported"
   - Contiene `timeout`/`Timeout` → "Connection timeout"
   - Contiene `NOT_FOUND` + `selector` → "Selector not found"
   - Contiene `HTTP_ERROR` → "HTTP error"
   - Altrimenti: prime 60 chars + "…"
3. Renderizzare `result.summary ?? result.detail` inline, `result.detail` nel tooltip

**File**: `ProviderAssignmentSection.svelte`

## §6 — Tooltip: doppio-click per copiare + toast

### Problema

L'utente vuole copiare il contenuto del tooltip al clipboard con doppio-click (desktop) o long-press (mobile), con toast blu di conferma.

### Soluzione

1. In `Tooltip.svelte`, sul div `.tooltip-fixed`:
   - `pointer-events: auto` (era `none`) — servono per intercettare click
   - `ondblclick` → `navigator.clipboard.writeText(text || tooltipElement.innerText)` + toast info
   - Per mobile: `ontouchstart` avvia timer 500ms, `ontouchend`/`ontouchmove` lo cancella. Se scade → copia
2. Toast via il toast store del progetto (importare da `$lib/stores/toasts`)
3. Flash visivo momentaneo (border color change per 300ms)

**File**: `Tooltip.svelte`

## §7 — Late interest row: checkbox non selezionabile

### Problema

La riga late interest mostra un checkbox cliccabile. L'utente ci clicca anche se poi viene filtrato.

### Root cause

`DataTable.svelte` renderizza checkbox per ogni riga senza possibilità di esclusione. `handleSelectionChange` filtra post-hoc ma il feedback visivo è errato.

### Soluzione

2. Nel template checkbox (L975): se `isRowSelectable?.(row) === false`, renderizzare `<td class="td-fixed td-select"><div style="width:28px"></div></td>` come placeholder
3. Escludere righe non-selezionabili da `toggleAllPageRows()`
4. In `ScheduledInvestmentEditor.svelte`: `isRowSelectable={(row) => !row.isLate}`

**File**: `DataTable.svelte`, `ScheduledInvestmentEditor.svelte`

## §8 — Grace period: input y/m/d con conversione

### Problema

Il grace period accetta solo giorni come numero. L'utente vuole inserire anni, mesi e giorni separatamente.

### Soluzione

1. In `CellDateRange.svelte`, nel grace popover, sostituire il singolo input con 3 input:
   ```
   [__] y  [__] m  [__] d    (= N days)
   ```
2. Conversioni:
   - `daysToYMD(total)`: `y = floor(total/365)`, `m = floor((total%365)/30)`, `d = total%365%30`
   - `ymdToDays(y,m,d)`: `y*365 + m*30 + d`
3. State locali: `localY`, `localM`, `localD` inizializzati da `daysToYMD(graceDays)` via `$effect`
4. Su blur di qualsiasi input: ricalcolo → `onGraceDaysChange(ymdToDays(...))`
5. Frecce su/giù mantengono il controllo sui giorni totali

**File**: `CellDateRange.svelte`
