# F-049 — BRIM Import UI asset matching wizard

## What it does
The frontend flow for importing broker reports via BRIM. Users upload a file, select or auto-detect the plugin, review the parsed transactions in a staging area, resolve unknown assets (mapping fake negative IDs to real assets), and confirm the import. The matching wizard shows extracted asset hints (ISIN, ticker, name) from the parsed file and lets users link each to an existing asset or create a new one.

## Layer breakdown
**Frontend**: `frontend/src/lib/components/brokers/` (BrokerImportFilesModal, AssetMatchingWizard), `frontend/src/routes/(app)/brokers/[id]/` — broker detail with import tab

## Dependencies
- [[F-012]] BRIM Framework — parse API
- [[F-013]] BRIM Plugins — 11 parsers providing `extracted_assets` hints
- [[F-011]] File Management — file uploaded first

## Enables
- [[F-048]] Staging Modal — matched transactions forwarded to staging

## Implementation notes
- Flow: Upload (F-011) → Select plugin → `POST /import/files/{id}/parse` → Review staging → Asset matching → Forward to Staging Modal
- `extracted_assets` from parse response: list of `BRIMExtractedAssetInfo` (ISIN, ticker, name, fake_id)
- `GET /import/files/{id}/last-parse` — reopens staging without re-parsing
- `search_asset_candidates()` service: searches local DB + provider search for each extracted asset
- `detect_tx_duplicates()` service: warns about transactions already in DB (same broker/date/asset/amount)
