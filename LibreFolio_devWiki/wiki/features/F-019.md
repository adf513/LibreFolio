# F-019 — MANUAL Sentinel FX Provider

## What it does
A special FX provider that acts as a sentinel for currency pairs not covered by any real central-bank provider. It allows users to enter exchange rates manually for exotic pairs.

## Layer breakdown
**Backend**: `backend/app/services/fx_providers/manual.py`

No frontend-specific component — the MANUAL provider is transparent to the UI and hidden from the public provider list.

## Dependencies
- [[F-015]] FX Provider Registry — registered via same pattern
- [[F-016]] FX Pair CRUD — activated/deactivated based on pair's provider assignments

## Enables
- [[F-017]] FX Rate Sync — MANUAL pairs still participate in sync (user enters rates manually)

## Implementation notes
- **Auto-insert**: when a pair has no real provider assigned, MANUAL is automatically added with `priority=999`
- **Auto-remove**: when a real provider is added to the pair, MANUAL is removed
- **Auto-reinstate**: when the last real provider is removed, MANUAL is re-added
- Hidden from `GET /api/v1/fx/providers` response — users never explicitly assign MANUAL
- MANUAL rates are not fetched — they are entered via the manual rate editor in the FX detail page
