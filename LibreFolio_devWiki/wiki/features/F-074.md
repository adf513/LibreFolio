# F-074 — E2E Test Gallery auto-screenshots

## What it does
Automated UI screenshot generation for the MkDocs documentation gallery. A dedicated Playwright test file (`frontend/e2e/gallery.spec.ts`) captures screenshots of key UI states at both desktop and mobile viewports, in light and dark themes, in all four languages (EN/IT/FR/ES). Screenshots are saved to `mkdocs_src/docs/gallery/` organized by `{viewport}/{lang}/{theme}/{category}/{name}.png`.

## Layer breakdown
**Infra**: `frontend/e2e/gallery.spec.ts` — screenshot capture spec, `mkdocs_src/docs/gallery/` — output directory, `mkdocs_src/docs/javascripts/` (gallery loader JS resolves image paths in docs pages)

## Dependencies
- [[F-067]] Playwright E2E Tests — gallery uses the same test infrastructure
- [[F-069]] MkDocs Documentation — screenshots embedded in docs pages

## Enables
- (visual documentation — no downstream features)

## Implementation notes
- Run: `./dev.py test gallery` or `./dev.py mkdocs gallery`
- Viewport projects: `desktop` (1280×720) and `mobile` (iPhone 14 Pro Max)
- Theme loop: test visits each page in both `light` and `dark` modes
- Language loop: sets locale via `setLanguage()` helper before each screenshot
- Gallery image path convention in docs: `{basePath}/gallery/{viewport}/{lang}/{theme}/{category}/{name}.png`
- `<img class="gallery-img" data-category="..." data-name="...">` — resolved by `gallery-loader.js`
