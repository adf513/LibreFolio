# F-054 — Dashboard KPI & Overview

## What it does
The main dashboard at `/dashboard` showing portfolio-level KPI cards (NAV, PnL, ROI), recent transactions widget, and quick action buttons. NAV and PnL are aggregated using `share_percentage` from `BrokerUserAccess` — each broker's values are weighted by the user's ownership share. Warning is shown if Σ(share%) > 100%.

New backend endpoint needed: `GET /api/v1/portfolio/overview` for weighted aggregation.

## Layer breakdown
**Backend**: Planned — `GET /api/v1/portfolio/overview`, aggregates broker NAV × `share_percentage`

**Frontend**: `frontend/src/routes/(app)/dashboard/+page.svelte` (currently placeholder), planned components: `KPICard.svelte`, `RecentTransactions.svelte`, `QuickActions.svelte`

## Implementation notes
- `KPICard` shows NAV/PnL/ROI with `share_percentage` weighting — NEVER averaging percentages
- EDITOR with share 0% → broker excluded from Net Worth
- `AnalyticsMethodSelect`: FIFO/LIFO/PMC dropdown — affects analytical view, always shows PMC disclaimer
- `RecentTransactions`: last 10 tx with compact DataTable
