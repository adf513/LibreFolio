# F-002 — User Management admin CRUD

## What it does
Admin-only CRUD for managing application users. Admins can list users, create new accounts, update user details (username, email, role, active status), reset passwords, and deactivate accounts. Regular users can only update their own profile and password via `PATCH /auth/profile` and `POST /auth/change-password`.

## Layer breakdown
**Backend**: `backend/app/api/v1/users.py` (endpoints: GET `/users`, POST `/users`, GET `/users/{id}`, PATCH `/users/{id}`, DELETE `/users/{id}`), `backend/app/services/user_service.py`

**Frontend**: Settings page `frontend/src/routes/(app)/settings/` — user management tab visible only to admins

## Dependencies
- [[F-001]] Auth — all user management endpoints require authenticated admin
- [[F-003]] Role System — distinguishes admin vs user capabilities

## Enables
- (none beyond general app usability)

## Implementation notes
- Admin check uses `get_admin_user` dependency (role == `admin`)
- First registered user is auto-promoted to admin — no bootstrap step needed
- `is_active=False` disables login without deleting the account (soft disable)
- Deleting a user cascades to their settings but not to broker data (brokers are shared resources)
