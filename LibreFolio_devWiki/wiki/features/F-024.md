# F-024 — Asset CRUD

## What it does
Full CRUD for financial assets (stocks, ETFs, bonds, crypto, etc.). Assets store display name, currency, asset type, and classification params (sector, geography, etc.) as JSON. Asset type is one of: STOCK, ETF, BOND, CRYPTO, COMMODITY, FUND, FOREX, OPTION, FUTURE, REAL_ESTATE, SCHEDULED_YIELD, OTHER.

The `FAAssetCreateItem` schema accepts the asset profile plus optional initial provider assignment so both steps can be done in one API call.

## Layer breakdown
**Backend**: `backend/app/api/v1/assets.py` (POST `/assets`, GET `/assets/query`, GET `/assets/{id}`, PATCH `/assets/{id}`, DELETE `/assets` bulk), `backend/app/services/asset_source.py` (`AssetCRUDService`), `backend/app/db/models.py` (`Asset` model)

**Frontend**: `frontend/src/routes/(app)/assets/` — list + `[id]` detail pages, `frontend/src/lib/components/assets/AssetModal.svelte` (1328 lines — create/edit modal with provider auto-assign)

## Implementation notes
- `GET /assets/query` supports filters: `active`, `asset_type`, `currency`, `search` — returns `FAinfoResponse` list
- `classification_params` JSON blob: `{sector, country, isin, ...}` — flexible, not schema-enforced at DB level
- Bulk delete: `DELETE /assets` with list of IDs
- `Asset.active` flag: when `False`, scheduler skips sync (Phase 8)
