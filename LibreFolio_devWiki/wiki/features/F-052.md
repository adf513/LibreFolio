# F-052 — Market Data Scheduler

## What it does
An embedded background daemon within the FastAPI process that automatically keeps market data current: current-price refresh at a configurable interval (default 10 min) and daily history sync at a configurable time (default 23:00 server time). Settings are read from `GlobalSetting` on every tick — no restart needed to apply changes. Uses asyncio native loop (not APScheduler — see notes).

## Layer breakdown
**Backend**: Planned — new scheduler module in `backend/app/`, state file `backend/data/{env}/scheduler_state.json`, new endpoint `GET /api/v1/admin/scheduler/state`

## Dependencies
- [[F-006]] Global Settings — scheduler config stored here (`scheduler_enabled`, `scheduler_current_price_frequency_minutes`, `scheduler_history_sync_time`, `scheduler_history_sync_horizon_days`)
- [[F-027]] Asset Data Sync — scheduler calls existing sync service methods
- [[F-017]] FX Rate Sync — scheduler calls existing FX sync

## Enables
- [[F-053]] Scheduler Settings UI — UI to configure the scheduler

## Implementation notes
- Architecture: asyncio native loop, 1-min tick, reads settings from DB every tick
- Multi-worker: `psutil` + lowest-PID leader election re-evaluated each tick (self-healing, no stale locks)
- Decision: APScheduler **rejected** — overhead for just 2 jobs; `reschedule_job()` complexity unjustified
- State: `scheduler_state.json` written atomically (write-then-rename) — survives restart
- `Asset.active = False` and `FxPair.active = False` skipped by scheduler
- Timezone: server local time (Docker container TZ)
- New dependency: `psutil` for cross-platform process listing
