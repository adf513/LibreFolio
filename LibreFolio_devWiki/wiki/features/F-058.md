# F-058 — ROI Calculations (TWRR + MWRR)

## What it does
Server-side portfolio return calculations: **TWRR** (Time-Weighted Rate of Return, geometrically linked
sub-period returns — eliminates the distorting effect of cash flows) and **MWRR** (Money-Weighted Rate of
Return / XIRR via Newton-Raphson — accounts for cash-flow timing). See [[concepts/twrr-mwrr-algorithms]] for
the algorithms. FIFO/WAC cost-basis matching is computed at runtime by [[entities/portfolio-engine]], not
persisted.

## Layer breakdown
**Backend**: `backend/app/utils/roi_utils.py` (`calculate_twrr`, `calculate_mwrr`) — called from
`portfolio_service.py::get_summary()` / `get_history()`, surfaced via the unified `POST /portfolio/report`
endpoint (see [[concepts/portfolio-report-unified]]).

## Dependencies
- [[F-046]] Transaction API — transaction history is the input for cash-flow / cost-basis calculations
- [[F-057]] Currency Conversion — all values converted to base currency before aggregation

## Enables
- [[F-054]] Dashboard KPI — TWRR/MWRR shown in the Weighted ROI KPI card

## Implementation notes
- MWRR/XIRR uses **Newton-Raphson only** (no Brent/hybrid fallback) with a result cap — a deliberate design
  choice, not an open bug. See [[decisions/mwrr-solver-newton-cap]].
- A double-counting boundary bug (synthetic `t=0` flow duplicating deposits already in `cash_flows_perf`) was
  fixed 2026-06-30 — see [[decisions/mwrr-boundary-fix]].
- PMC (average cost / Prezzo Medio di Carico): analytical view only, always with disclaimer "not tax-legal in
  most jurisdictions".
- Fiscal regimes (tax-specific FIFO/LIFO rules): postponed beyond Phase 9 (see [[F-081]]).
