# F-070 — Aphra LLM Translation Pipeline

## What it does
A custom LLM-powered pipeline for translating MkDocs documentation from English to Italian, French, and Spanish. Named "Aphra" (after Aphra Behn). Reads `.en.md` source files, sends them to an LLM via structured prompts from `mkdocs_src/aphra-pipeline/prompts/`, and writes `.it.md`, `.fr.md`, `.es.md` output files. Includes validation to detect structural drift (missing headings, broken links) between source and translations.

## Layer breakdown
**Infra**: `mkdocs_src/aphra-pipeline/translate_docs.py` (main pipeline), `mkdocs_src/aphra-pipeline/validate_translations.py` (drift checker), `mkdocs_src/aphra-pipeline/prompts/` (LLM prompts), `mkdocs_src/aphra-pipeline/glossaries/` (domain-specific terminology)

## Implementation notes
- Triggered via `./dev.py mkdocs translate`
- Post-translation validation: `./dev.py mkdocs translate-diff --issues-only` — reports structural differences
- Glossaries: domain-specific terms (financial, technical) to keep consistent across translations
- Large changes → run pipeline; small changes or targeted fixes → edit translations directly with AI agent
- Developer docs (`developer/`) are EN-only — pipeline skips them by design
