# F-068 — Backend API Tests pytest

## What it does
Backend test suite with 850+ tests across 8 categories using pytest + `httpx` async client. Tests cover REST endpoints, database layer, business logic services, external provider calls, end-to-end backend flows, schema validation, and utilities. Tests are isolated per-user via `unique_id()` helper — no shared state between tests.

## Layer breakdown
**Infra**: `backend/test_scripts/` — organized by category: `test_api/` (276+ tests), `test_db/`, `test_services/`, `test_e2e/`, `test_external/`, `test_schemas/`, `test_utilities/`, `test_server_helper.py` (`_TestingServerManager`), `test_utils.py`

## Dependencies

## Implementation notes
- Pattern: `@pytest.mark.asyncio` class with `autouse` fixture starting `_TestingServerManager`
- `_TestingServerManager`: starts backend server in a thread on `TEST_PORT` for API tests
- `API_BASE = f"http://localhost:{settings.TEST_PORT}/api/v1"`, `TIMEOUT = 30`
- Test isolation: each test creates its own user via `unique_id()` + `create_user_and_login()`
- `print_section()` / `print_success()` helpers for readable test output
- Run: `./dev.py test api|db|services|external|e2e|all-backend`
- `./dev.py test --coverage api all` — pytest-cov coverage tracking
- External tests (providers): require network — excluded from CI by default
