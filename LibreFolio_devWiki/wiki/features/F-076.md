# F-076 — Log Level Policy & TRACE Level

## What it does
Formalizes the backend logging policy with a custom `TRACE=5` level (below DEBUG), structlog extension for structured JSON logging, and a policy document in `logging_config.py` that defines what goes at each level. Full audit of all `logger.*` calls was performed to align with the policy.

## Implementation notes
- `TRACE = 5` registered in stdlib logging, structlog `LEVEL_TO_NAME`, and Logger class (`.trace()` method)
- 6-level policy documented in header comment of `logging_config.py`
- Practical rule: "user did X"→INFO, "system decided X"→DEBUG, "value X for date Y repeated N times"→TRACE
- Bug fix: structlog 25.5.0 `KeyError: 5` resolved by patching `LEVEL_TO_NAME[5] = "trace"`
- Full audit downgraded verbose INFO→DEBUG in providers, cache_utils, etc.
- Completed: 2026-06-01
