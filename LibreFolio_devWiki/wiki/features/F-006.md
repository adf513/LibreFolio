# F-006 — Global Settings admin-managed

## What it does
System-wide settings managed by admins and visible to all users. Stored in the `global_setting` table as key/value pairs with a typed `type` column. Current keys include `session_ttl_hours`, `portfolio_base_currency`, and (planned Phase 8) scheduler settings. The `GlobalSettingsTab.svelte` renders these settings in the admin section of the settings page.

## Layer breakdown
**Backend**: `backend/app/api/v1/settings.py` (GET `/settings/global`, PATCH `/settings/global` — admin only), `backend/app/services/global_settings_service.py`, `backend/app/db/models.py` (`GlobalSetting` model: `key`, `value`, `type`)

**Frontend**: `frontend/src/lib/stores/globalSettings.ts`, settings page admin tab

## Dependencies
- [[F-003]] Role System — only admins can write global settings

## Enables
- [[F-052]] Market Data Scheduler — scheduler settings (`scheduler_enabled`, `scheduler_current_price_frequency_minutes`, `scheduler_history_sync_time`, `scheduler_history_sync_horizon_days`) stored here

## Implementation notes
- `GlobalSetting` model: `key` (unique), `value` (string), `type` enum (bool/int/string/float)
- `get_session_ttl_hours()` helper used by auth to read TTL dynamically
- `PORTFOLIO_BASE_CURRENCY` env var overrides the DB setting at startup
- Admins read/write via `GET/PATCH /api/v1/settings/global`
