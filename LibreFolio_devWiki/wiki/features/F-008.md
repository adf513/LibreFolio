# F-008 — i18n System EN/IT/FR/ES

## What it does
Full frontend internationalization supporting English, Italian, French, and Spanish. All UI strings are externalized to JSON translation files with 840+ keys per language. The active language is persisted in user settings and applied at startup. New keys are managed via the `./dev.py i18n` CLI which handles audit, add, remove, update, and search operations.

## Layer breakdown
**Frontend**: `frontend/src/lib/i18n/{en,it,fr,es}.json` (translation files), `frontend/src/lib/stores/language.ts` (active language store), svelte-i18n library, `frontend/src/lib/components/layout/LanguageSelector.svelte`

## Implementation notes
- Library: `svelte-i18n`
- Key count: 840+ per language (as of Phase 6)
- Namespaces: flat with dot notation (e.g. `assets.list.title`, `signals.ema`)
- CLI: `./dev.py i18n audit` detects missing keys across languages
- `./dev.py i18n add <key> <en-value>` adds a key to all 4 language files
- Flag emojis use `Noto Color Emoji` font loaded separately
