# F-067 — Playwright E2E Tests

## What it does
End-to-end frontend test suite using Playwright, covering all major user flows: auth, settings, broker management, FX pairs, asset management, file operations. 181+ tests across two browser projects (desktop 1280×720 and mobile iPhone 14 Pro Max viewport), both using Chromium.

## Layer breakdown
**Infra**: `frontend/e2e/` — test files organized by domain, `frontend/e2e/fixtures/` (auth-helpers, db-helpers, test-users, i18n-data), `playwright.config.ts`

## Dependencies
- [[F-064]] Data Separation — tests run against isolated test DB

## Enables
- [[F-074]] E2E Test Gallery — gallery screenshots generated during E2E runs

## Implementation notes
- Workers: 1 (sequential — shared DB state between tests)
- Test server: auto-started via `./dev.py server --test --force` in web server config
- Conventions: `data-testid` attributes always (never CSS classes or text — fragile with i18n)
- `login()`, `logout()`, `navigateTo()` helpers from `auth-helpers.ts`
- `resetDatabase()`, `populateDatabase()` from `db-helpers.ts`
- `openMobileMenu()` for mobile viewport hamburger handling
- Run commands: `./dev.py test front-utility|front-user|front-fx|front-asset|all-frontend`
- WebKit excluded (stability issues on Linux)
