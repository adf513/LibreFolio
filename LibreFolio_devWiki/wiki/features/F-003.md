# F-003 — Multi-User Role System

## What it does
Two-level role system: **admin** (full access including user management and global settings) and **user** (access to their own data and shared brokers they have been granted access to). At the broker level, a finer-grained Owner/Editor/Viewer model ([[F-010]]) controls per-resource access.

## Layer breakdown
**Backend**: `backend/app/db/models.py` (`UserRole` enum: `admin`, `user`; `BrokerUserAccess` model), `backend/app/api/v1/auth.py` (`get_admin_user` dependency), `backend/app/services/user_service.py`

**Frontend**: role-aware UI — admin-only tabs in settings, conditional rendering based on `$auth` store

## Dependencies
- [[F-001]] Auth — role is attached to the authenticated user record

## Enables
- [[F-010]] Broker Sharing — the Owner/Editor/Viewer sub-system extends the role model to broker resources

## Implementation notes
- `UserRole` is a `StrEnum` stored as a text column in the `user` table
- `get_admin_user` dependency: calls `get_current_user` then checks `user.role == "admin"`, raises 403 otherwise
- Role change is admin-only via `PATCH /users/{id}`
- No per-asset or per-FX-pair ACL — only broker-level ACL via `BrokerUserAccess`
