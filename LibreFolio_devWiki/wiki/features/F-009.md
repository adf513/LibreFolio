# F-009 — Broker CRUD

## What it does
Full CRUD for investment brokers (brokerage accounts). A broker is the top-level container for transactions and files in LibreFolio. Users create brokers to represent their accounts (e.g., "IBKR Main", "Degiro"), configure access control, and associate imported transactions and files with them.

The broker summary endpoint provides aggregated data: balances, holdings, and transaction counts per broker.

## Layer breakdown
**Backend**: `backend/app/api/v1/brokers.py` (POST `/brokers`, GET `/brokers`, GET `/brokers/{id}`, GET `/brokers/{id}/summary`, PATCH `/brokers/{id}`, DELETE `/brokers` bulk), `backend/app/services/broker_service.py`, `backend/app/db/models.py` (`Broker` model: `name`, `description`, `icon_url`, `owner_id`)

**Frontend**: `frontend/src/routes/(app)/brokers/` — list page + `[id]` detail page, `frontend/src/lib/components/` broker-specific components

## Dependencies
- [[F-003]] Role System — brokers are owned by a user, with shareable access

## Enables
- [[F-010]] Broker Sharing — access control per broker
- [[F-011]] File Management — files scoped to broker
- [[F-012]] BRIM Framework — import files uploaded per broker
- [[F-014]] Image Upload & Crop — broker icon upload

## Implementation notes
- Bulk delete: `DELETE /brokers` with list of IDs
- `owner_id` set to the creating user's ID; owner always has OWNER access in `BrokerUserAccess`
- `GET /brokers` filtered by the calling user's `BrokerUserAccess` entries — users only see brokers they have access to
