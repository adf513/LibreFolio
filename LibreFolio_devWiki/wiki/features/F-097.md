# F-097 — WAC (Weighted Average Cost)

## What it does
Calculates the weighted average cost basis for assets using all historical BUY prices and incoming TRANSFER cost_basis values, with automatic FX conversion for multi-currency portfolios. Auto-fills `cost_basis_override` on TRANSFER receiver transactions when no explicit value is provided. Provides a recalculation endpoint for bulk WAC refresh.

## Layer breakdown
**Backend**:
- `compute_weighted_avg_cost(session, broker_id, asset_id, as_of_date, asset_currency)` → `WACResult`
- FX cross-currency: converts each BUY/TRANSFER amount via FX rate at TX date
- Target currency determination: majority → asset currency on tie → alphabetical
- Auto-calc at commit: applied to TRANSFER receiver (qty>0) when `cost_basis_override` is null
- `POST /transactions/recalc-wac` — bulk recalculation for specified TX IDs (same-asset constraint)
- `cost_basis_override` upgraded from `SafeDecimal` to `Currency` object (`{code, amount}`)
- `WACResult` schema: `wac: Optional[Currency]`, `conversions: list[WACConversionInfo]`, `missing_pairs: list[str]`
- `wac_info: Optional[WACResult]` on `TXBatchResultItem` (commit/promote response only)

**Frontend**:
- `WacPreviewSection.svelte` — controlled component (mode from parent, no internal state) with Auto/Manual toggle
- BulkModal: fingerprint-based invalidation triggers WAC recalc only on material field changes
- BulkModal: `fxMarketCache` for FX rate display, `wacResults` Map for cell display
- FormModal: WAC preview with qualifying transaction table, pending indicator (●), apply/discard flow
- `CostBasisFieldMode` state machine: `auto | manual | auto-detail` (per-row, not persisted to DB)
- CompactCash component for cost_basis input (amount + currency selector)
- Warning banner for ADJUSTMENT qty>0 without cost_basis_override
- FX implied rate + market spread display in suggest banner (`lookupFxRate()` in `fxStoreRegistry.ts`)

## Implementation notes
- **Breaking change**: old `"42.50"` string format for cost_basis → 422 (no backward compatibility)
- **Fail-safe**: if even one FX conversion is impossible → `WACResult(wac=None)`, TX gets `cost_basis_override=null`
- **Zero-quantity edge case**: `total_qty == 0` with no FX errors → `Currency(code=target, amount="0")`
- **WAC inline in validate/commit** ([[decisions/wac-inline-validate-commit]]): no standalone endpoint in editing flow. Preview via /validate response, apply via /commit post-flush
- **cost_basis_mode**: explicit `'auto' | 'manual'` field in payload — prevents feedback loop ([[problems/wac-feedback-loop]])
- **`add_at_wac` semantics**: TRANSFER_IN with auto mode adds quantity at current WAC — algebraically proven to not change WAC
- **link_uuid promotion**: top-level on PendingOp, not create-only — fixes clone bug ([[problems/clone-link-uuid-duplication]])
- **Stateless preview** ([[concepts/stateless-preview-pattern]]): WacPreviewSection is controlled (mode as prop), cache on PendingOp._wacCache
- **Partner row WAC**: wac_results indices remapped to visible ops using operation field + positional matching
- **cost_basis_override semantics**: applied ONLY to receiver (qty>0); sender (qty<0) forced to null
- **Backward-fill unlimited**: `get_prices_bulk` uses unlimited backward-fill (like FX system) — no artificial 30-day cutoff

## Dead ends / Ideas explored
- Initially considered keeping backward compat with bare decimal string — rejected because Currency object provides unambiguous currency association
- Considered persisting `wac_info` to DB — rejected because it's a derived value that changes when FX rates or prior transactions change
- **Standalone `/wac-preview` endpoint**: caused feedback loops (result → field change → re-request). Replaced with WAC inline in validate/commit
- **`null` as auto sentinel**: using `cost_basis_override = null` to mean "auto" was ambiguous. Replaced with explicit `cost_basis_mode` field
- **ReactiveWacBulkModal v1-v3**: initial designs (Steps 1-8) failed due to reactivity issues. Rearchitected via v4-v6 plans before stable
- **WacPreviewSection with internal state**: caused mode loss on re-open due to `untrack()` timing. Replaced with fully controlled component pattern
