# F-026 — Asset Provider Assignment dynamic params

## What it does
Assigns data providers to assets with provider-specific configuration parameters. Each provider exposes a `params_schema` that drives a dynamic form in the frontend — the form fields are generated at runtime based on the provider's declared `SignalParamDescriptor`-like schema. For example, CSS Scraper requires `current_css_selector` and `currency`, while yfinance requires only a ticker symbol.

## Layer breakdown
**Backend**: `backend/app/api/v1/assets.py` (POST `/assets/provider/assign`, DELETE `/assets/provider` bulk, GET `/assets/provider/list`), `backend/app/db/models.py` (`AssetProviderAssignment`: `asset_id`, `provider_code`, `identifier`, `identifier_type`, `provider_params` JSON)

**Frontend**: `frontend/src/lib/components/assets/ProviderAssignmentSection.svelte` (724 lines), `frontend/src/lib/components/assets/AssetModal.svelte` — integrated in create/edit modal

## Implementation notes
- `AssetProviderAssignment.provider_params` is a JSON blob validated by the provider's `params_schema` at assignment time
- `IdentifierType` values: TICKER, ISIN, URL, UUID (auto-generated)
- `ProviderInputType` enum tells the frontend what input to show: TICKER/ISIN/URL/AUTO_GENERATED
- `FAProviderProbeRequest`/`FAProviderProbeResponse` — dry-run test of configuration before saving
- Multiple providers can be assigned to one asset (e.g., yfinance for prices + scheduled_investment for events)
