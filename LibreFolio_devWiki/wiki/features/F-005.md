# F-005 — User Settings language/theme

## What it does
Per-user persistent settings stored server-side in the `user_settings` table. Covers UI language (EN/IT/FR/ES) and theme preference (light/dark/auto). Settings are loaded on login and applied before the first render, preventing flash of wrong theme/language.

## Layer breakdown
**Backend**: `backend/app/api/v1/settings.py` (GET/PATCH `/settings/me`), `backend/app/services/settings_service.py`, `backend/app/db/models.py` (`UserSettings` model — JSON blob per user)

**Frontend**: `frontend/src/lib/stores/settings.ts`, settings page `frontend/src/routes/(app)/settings/` — personal preferences tab, `frontend/src/lib/stores/language.ts`, `frontend/src/lib/stores/themeStore.ts`

## Dependencies
- [[F-001]] Auth — settings scoped to authenticated user
- [[F-007]] Theme System — theme preference applied at store level
- [[F-008]] i18n — language preference applied at i18n store level

## Enables
- (no explicit feature dependencies — baseline user experience)

## Implementation notes
- `UserSettings` stores a JSON blob; frontend merges with defaults for missing keys
- Language and theme are read from server settings on load, written on change
- `themeStore` key in localStorage: `librefolio-theme` (also backed server-side)
- API: `GET /api/v1/settings/me`, `PATCH /api/v1/settings/me`
