# F-055 — Portfolio Charts (Holdings/Performance panel, GrowthChart, Allocation)

## What it does
ECharts-based portfolio visualization on the Dashboard:
- **GrowthChart** — NAV / invested capital / cash 3-pool stacked-area chart with rich tooltip (see [[concepts/3-pool-cash-model]])
- **AllocationPanel** — Now/History toggle; `AllocationHistoryChart.svelte` shows allocation evolution over time across 3 dimensions (type/sector/geography)
- **Holdings/Performance panel** (`PositionsPanel.svelte`) — Table/Chart toggle per tab: `ExposureTreemap.svelte` (Holdings, area=value/color=P&L%) and `PerformanceChart.svelte` (Performance, stacked diverging bar). See [[concepts/holdings-performance-panel]] for the full refactor (renamed from Exposure/Contribution, commit `78aaa0a3`).

## Layer breakdown
**Frontend**: `frontend/src/lib/components/dashboard/` — `GrowthChart.svelte`, `AllocationHistoryChart.svelte`, `AllocationPanel.svelte`, `PositionsPanel.svelte`, `ExposureTable.svelte`, `ExposureTreemap.svelte`, `ContributionTable.svelte`, `PerformanceChart.svelte`, `OtherPeriodEffectsTable.svelte`, `KpiSection.svelte`. Shared treemap zoom/pan guard: `frontend/src/lib/components/charts/echartsTreemapZoomGuard.ts`.

## Dependencies
- [[F-054]] Dashboard KPI — charts live on the dashboard page, share the same `/portfolio/report` fetch
- [[F-046]] Transaction API — data source for gain/loss calculations

## Enables

## Implementation notes
- All charts derive from a single `portfolioStore.fetchReport()` call — no per-chart API round trip
- `ExposureTreemap`/`PerformanceChart` both use `echartsTreemapZoomGuard.ts` (Holdings treemap only) for zoom/pan clamping — fixes a bug where ECharts' internal `_resetController` let cumulative zoom-out shrink the treemap to a dot and pan drag it off-screen; the guard listens to both `treemaprender` and `treemapmove` chart events (verified via `TreemapView.js` source)
- `PerformanceChart` splits stack by sign (`positive`/`negative`), not a single linear stack; costs always rendered as negative regardless of DTO sign convention
- `ContributionTreemap.svelte` was removed entirely, replaced by `PerformanceChart.svelte`
- The global Open/Closed toggle was removed — Status is now a native DataTable enum filter column in the Performance table
- Known unrelated bug: DataTable column-resize handle is a no-op in some tables — see [[problems/datatable-column-resize-noop]] (does not block chart functionality)
