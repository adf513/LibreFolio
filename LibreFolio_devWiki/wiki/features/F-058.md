# F-058 — ROI Calculations Simple/DWR

## What it does
Server-side portfolio return calculations using FIFO cost-basis matching (computed at runtime, not persisted). Supports Simple ROI (total return on cost basis) and Dollar-Weighted Return (DWR/IRR — accounts for timing of cash flows). FIFO matching is the primary method; LIFO and PMC (average cost) are planned as analytical alternatives shown with a disclaimer.

## Layer breakdown
**Backend**: Planned — `backend/app/services/` — new portfolio analytics service, `GET /api/v1/portfolio/overview` (see F-054)

## Dependencies
- [[F-046]] Transaction API — transaction history is the input for cost-basis calculations
- [[F-057]] Currency Conversion — all values converted to base currency before aggregation

## Enables
- [[F-054]] Dashboard KPI — ROI shown in KPI cards

## Implementation notes
- **FIFO at Runtime**: cost matching computed on-demand from transaction history — not persisted in DB (fundamental architecture decision)
- Simple ROI: `(CurrentValue - TotalCost) / TotalCost × 100`
- DWR/IRR: accounts for timing of BUY/SELL transactions, uses Newton-Raphson for IRR solve
- PMC (average cost): analytical view only, always with disclaimer "not tax-legal in most jurisdictions"
- Fiscal regimes (tax-specific FIFO/LIFO rules): postponed beyond Phase 9
