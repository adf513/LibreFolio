# F-082 — Cash Split Transactions

## What it does
A UI feature for splitting a cash transaction (BUY/SELL) into multiple sub-records. Includes a support table for split line items, a UI box showing split details with bound totals validation (sum of splits must equal original amount).

## Implementation notes
- Requires a support table for split items (or JSON column on Transaction)
- UI: expandable split box in the Staging Modal with constrained totals
- Medium priority
