# F-004 — App Shell & Navigation

## What it does
The persistent application shell wrapping all authenticated pages. Includes a collapsible sidebar with navigation links (Dashboard, Brokers, FX, Assets, Transactions, Files, Settings), a top header bar with theme toggle and language selector, and a footer. On mobile, the sidebar is hidden behind a hamburger menu. The shell enforces authentication — unauthenticated users are redirected to login.

## Layer breakdown
**Frontend**: `frontend/src/routes/(app)/+layout.svelte` (auth guard + shell wrapper), `frontend/src/lib/components/layout/` (Header, Sidebar, Footer, LanguageSelector), `frontend/src/lib/stores/navigationStore.ts` (sidebar open/closed state)

## Dependencies
- [[F-007]] Theme System — dark/light toggle in header
- [[F-008]] i18n System — language selector in header, all nav labels translated

## Enables
- (all frontend pages — the shell is the container for everything)

## Implementation notes
- Auth guard in `+layout.ts` redirects to `/login` if no session cookie
- Sidebar collapse state persisted in `navigationStore` (Svelte writable)
- Mobile: hamburger menu with `openMobileMenu()` helper used in E2E tests
- `lucide-svelte` for all nav icons
- Brand color `#1a4031` (libre-green) used in sidebar header
