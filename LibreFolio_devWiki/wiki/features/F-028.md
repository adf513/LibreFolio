# F-028 — Asset Search Autocomplete

## What it does
Multi-provider parallel search component for finding assets by name, ticker, or ISIN. Queries all providers that implement `search()` (yfinance, JustETF) simultaneously and merges results deduplicated by identifier. Selecting a result auto-fills the asset creation form (name, currency, type) and pre-configures the provider assignment.

## Layer breakdown
**Frontend**: `frontend/src/lib/components/assets/AssetSearchAutocomplete.svelte` (399 lines), integrated in `AssetModal.svelte`

**Backend**: `backend/app/api/v1/assets.py` (`GET /assets/provider/search?q=...`), `backend/app/services/asset_source.py` (`AssetSearchService`)

## Dependencies
- [[F-025]] Asset Provider Registry — providers queried in parallel
- [[F-024]] Asset CRUD — search feeds into asset creation

## Enables
- (UX improvement for asset creation — no new data model)

## Implementation notes
- Results from all `search()`-capable providers merged client-side; deduplication by identifier
- Search results cache: 15 min (exact query) + 24h per individual result item
- `GET /assets/provider/search` dispatches to all providers in parallel threads
- Auto-fill: provider code + identifier + display_name + currency populated from selected result
