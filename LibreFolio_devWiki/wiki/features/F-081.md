# F-081 — Fiscal Sale Method (FIFO/LIFO/PMC/SelectID)

## What it does
Allows users to choose the fiscal cost-basis method per sale: FIFO (default), LIFO, PMC (average cost), or SelectID (pick specific lots). This is a high-complexity architectural change requiring: per-broker settings, per-user preference, global admin default, and a `link_transactions_id` field for SelectID lot tracking.

## Implementation notes
- Currently only FIFO is implemented at runtime ([[F-056]])
- New methods needed: LIFO (reverse chronological), PMC (weighted average), SelectID (lot-specific)
- **Broker settings**: override method per broker
- **User preference**: default method for all sales
- **Global admin default**: fallback for new users
- `link_transactions_id`: FK on `Transaction` to link a SELL to specific BUY lots (SelectID)
- High complexity — requires DB migration, service rewrite, and UI for lot selection modal
- HIGH priority but architecturally complex
