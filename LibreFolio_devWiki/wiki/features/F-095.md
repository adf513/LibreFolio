# F-095 — Asset Delete — Transaction Count Link

## What it does
When a user attempts to delete an asset that has linked transactions, the delete confirmation dialog shows the count of linked transactions and a link to `/transactions?asset_id=123`. This prevents accidental data loss and provides a quick path to review before deleting.

## Implementation notes
- Backend: `GET /assets/{id}` or delete endpoint should return `linked_transaction_count`
- Frontend: delete confirmation modal shows count + link if count > 0
- Requires Phase 7 transaction model to be live
