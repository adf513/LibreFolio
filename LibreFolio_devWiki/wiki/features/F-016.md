# F-016 — FX Pair CRUD

## What it does
CRUD for currency conversion pairs (e.g., EUR/USD, GBP/CHF). Each pair has a base currency, a quote currency, one or more provider assignments with priority ordering, and an `active` flag. When a pair is created, the system auto-assigns a suitable provider based on the base/quote currencies, and auto-inserts the MANUAL sentinel if no real provider covers the pair.

## Layer breakdown
**Backend**: `backend/app/api/v1/fx.py` (POST `/fx/routes`, GET `/fx/routes`, DELETE `/fx/routes` bulk, PATCH `/fx/routes/{id}`, GET `/fx/currencies`), `backend/app/db/models.py` (`FxConversionRoute`: `base`, `quote`, provider assignments + priority)

**Frontend**: `frontend/src/routes/(app)/fx/` — FX list page, create/edit modals, `frontend/src/lib/components/fx/`

## Dependencies
- [[F-015]] FX Provider Registry — providers assigned to pairs
- [[F-019]] MANUAL Sentinel — auto-inserted when no real provider covers the pair

## Enables
- [[F-017]] FX Rate Sync — pairs are the sync units
- [[F-021]] FX List View — displays configured pairs
- [[F-022]] FX Detail Page — per-pair detail
- [[F-023]] FX Chart — chart for a pair's rate history

## Implementation notes
- `FxConversionRoute` stores base/quote + provider assignment list with `priority` (lower = higher priority)
- MANUAL provider auto-inserted at `priority=999` when pair has no real provider
- `GET /fx/currencies` returns all ISO 4217 currencies + crypto supported by the system
- Bulk delete: `DELETE /fx/routes` with list of pair slugs
