# F-071 — EditBuffer Pattern

## What it does
A generic bidirectional edit buffer for pending chart/data modifications before bulk save. Manages `PendingEdit<T>` objects in three states: click-to-edit (chart interaction), CSV textarea (bulk text input), and form (manual entry). All three sync bidirectionally. On save, all pending edits are sent to the backend in a single bulk API call.

**Frontend**: `frontend/src/lib/stores/EditBuffer.ts` — generic class parameterized on `T extends TimeSeriesPoint`

## Implementation notes
- `PendingEdit<T>`: `{id, point, csvLineNumber, source, isNew}`
- `EditSource`: `'click' | 'csv' | 'form'`
- Bidirectional sync: CSV textarea parse → update buffer entries; chart click → add entry → re-render CSV
- `BufferChangeCallback<T>` — subscriber pattern for reactive updates
- Bulk save: collects all `PendingEdit.point` values → single `POST .../upsert` API call
- Idempotent: multiple edits to same date key collapse to last value
