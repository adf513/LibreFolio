# F-072 — TimeSeriesStore Pattern delta-fetching

## What it does
A generic client-side time-series cache using `Map<string, T>` keyed by ISO date. The core insight: when a chart component requests a date range, the store returns both the cached data and the **gaps** (contiguous missing intervals). The controller fetches only the gaps from the backend, then merges. This minimizes API calls for charts with overlapping or extending date ranges.

## Implementation notes
- `TimeSeriesPoint` constraint: `{date: string}` (ISO YYYY-MM-DD)
- `RangeResult<T>`: `{data: T[], gaps: DateGap[]}`
- `DateGap`: `{start: string, end: string}` — contiguous missing interval
- Merge: idempotent insert/update (same date key → last write wins)
- Invalidation: per-range clear or full cache reset (for manual refresh)
- Gap detection: O(N log N) using sorted date keys and binary search
