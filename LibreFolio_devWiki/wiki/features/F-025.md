# F-025 — Asset Provider Registry

## What it does
Registry of asset data source providers, each capable of fetching current prices, historical OHLCV data, and/or searching for assets. Providers are auto-discovered at startup. Four providers are currently active, plus a hidden mock provider for testing.

Providers:
- **yfinance** (`yahoo_finance.py`) — Yahoo Finance, ticker-based, supports search
- **justetf** (`justetf.py`) — JustETF, ISIN-based, supports search
- **cssscraper** (`css_scraper.py`) — CSS Web Scraper, URL-based, no search
- **scheduled_investment** (`scheduled_investment.py`) — deterministic engine for bonds/loans/P2P
- **mockprov** (`mockprov.py`) — testing only, hidden from UI

## Layer breakdown
**Backend**: `backend/app/services/asset_source_providers/`, `backend/app/services/asset_source.py` (abstract `AssetSourceProvider` base, `AssetSourceManager`, `AssetCRUDService`, `AssetSearchService`), `backend/app/services/provider_registry.py` (`AssetProviderRegistry`)

## Implementation notes
- Abstract base: `AssetSourceProvider` in `asset_source.py`
- `params_schema` property drives dynamic frontend forms — each provider declares its required parameters
- Thread isolation: each provider call runs in a dedicated thread with its own event loop via `_run_provider_in_thread()`
- `probe_provider_config()` endpoint bypasses cache for dry-run configuration testing
