# F-048 — Staging Modal

## What it does
A unified modal for staging transactions before atomic commit. Handles three input sources: manual entry (new transactions typed by user), BRIM import (parsed from broker report), and clone (copy of existing transactions). Features: asset-grouped rows with color coding, SearchSelect for bulk asset assignment, event-matching via tolerance slider (0–7 days), balance preview, and atomic per-broker commit via `POST /brokers/{id}/transactions/bulk`.

## Implementation notes
- "Staging area" concept: transactions held in component state, validated locally before server commit
- Asset resolver: unknown assets (from BRIM fake IDs) matched to real assets via `AssetMatchingWizard` or `AssetModal` create flow
- Tolerance slider (0–7 days): matches BRIM dividend transactions to existing `AssetEvent` records within the tolerance window
- Atomicity: single `POST /brokers/{id}/transactions/bulk` per broker — all succeed or all fail
- Phase 7 Part 5 — detailed design planned at execution time
