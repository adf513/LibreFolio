# F-047 — Transaction List Page

## What it does
The `/transactions` page showing all transactions the current user has access to across all their brokers. Uses the generic `DataTable` component with column header filters (date range, asset, broker, type, amount). Supports bulk actions (delete, edit, export). Clicking a transaction navigates to its asset detail or broker context.

## Layer breakdown
**Frontend**: `frontend/src/routes/(app)/transactions/+page.svelte` (currently a placeholder — Phase 7 Part 4), `frontend/src/lib/components/table/` (`DataTable` generic component)

## Dependencies
- [[F-046]] Transaction API — data source
- [[F-066]] Zodios API Client — typed API calls

## Enables
- [[F-048]] Staging Modal — accessible from this page for adding new transactions

## Implementation notes
- Design modeled after `AssetTable` / `FxTable` / `BrokerImportFilesModal` patterns
- Column header filters: date range (start/end pickers), asset (SearchSelect), broker (SimpleSelect), type (multi-select), amount range
- Consistent with other DataTable pages — same `DataTable` component, same filter UX
- "GoTo linked pair" — navigate to the asset or broker from a transaction row
- Phase 7 Part 4 target: list with filters + bulk actions
