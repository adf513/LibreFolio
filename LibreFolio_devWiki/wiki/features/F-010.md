# F-010 — Broker Sharing Owner/Editor/Viewer

## What it does
Fine-grained per-broker access control. The broker owner can share access with other users at three levels: **Owner** (full control including delete and resharing), **Editor** (can add/edit transactions and import files), **Viewer** (read-only access). Access entries are managed via the broker detail page.

## Layer breakdown
**Backend**: `backend/app/db/models.py` (`BrokerUserAccess` model: `broker_id`, `user_id`, `role`), `backend/app/api/v1/brokers.py` (access management endpoints: bulk create/update/delete on `/brokers/{id}/access`), `backend/app/services/broker_service.py`

**Frontend**: broker detail page sharing tab, `frontend/src/routes/(app)/brokers/[id]/`

## Dependencies
- [[F-009]] Broker CRUD — sharing applies to existing brokers
- [[F-003]] Role System — extends the app-level role model to broker resources

## Enables
- [[F-046]] Transaction API — `create_bulk` verifies `EDITOR` access before committing transactions

## Implementation notes
- `BrokerUserAccess.role` values: `owner`, `editor`, `viewer`
- `GET /brokers` only returns brokers where the user has an access entry
- `POST /brokers` auto-creates an OWNER entry for the creating user
- Dashboard share_percentage (planned Phase 9) — weighted NAV aggregation by `share_percentage` field in access entry
