# F-096 ŌĆö Scheduled Investment ŌĆö Decoupled Frequencies + Anchor Day

## What it does
Enhancement to the Scheduled Investment provider to decouple the frequency of price generation from the frequency of coupon maturation. Currently, `maturation_frequency` controls both, which means a bond with semiannual coupons can only show 2 price points per year on the chart. This feature would add:

1. **`price_frequency`** ŌĆö granularity of the price chart (e.g., DAILY)
2. **`coupon_frequency`** ŌĆö frequency of interest payout (e.g., SEMIANNUAL), only when `generate_interest=True`
3. **`coupon_anchor`** ŌĆö specify when the coupon falls (e.g., "1st of month", "specific day", "specific weekday")

## Layer breakdown
**Backend**: `backend/app/services/asset_source_providers/scheduled_investment.py` ŌĆö refactor `_generate_schedule_values()` to compute two separate date sets (price emission dates and coupon dates).

**Frontend**: `ScheduledInvestmentEditor.svelte` ŌĆö add UI fields for `price_frequency`, `coupon_frequency`, `coupon_anchor` with contextual validation.

## Dependencies
- [[F-034]] Scheduled Investment Provider ŌĆö base feature this extends

## Implementation notes
- **Schema**: `FAInterestRatePeriod` in `backend/app/schemas/assets.py` gets 3 new optional fields
- **Retrocompatibility**: If `price_frequency` is missing in existing JSON, default to `maturation_frequency` (no breaking change)
- **Engine**: Step 3 (emit value) uses `price_emission_dates`, Step 4 (coupon reset) uses `coupon_dates`
- **Anchor types**:
  - `SCHEDULE_START` (current behavior)
  - `FIRST_OF_MONTH` / `LAST_OF_MONTH`
  - `SPECIFIC_DAY_OF_MONTH(day: int)`
  - `SPECIFIC_WEEKDAY(weekday: str)`
  - `SPECIFIC_DATE(month: int, day: int)` for ANNUAL/SEMIANNUAL
