# F-020 — FX Currency Conversion Graph (Triangulation)

## What it does
Maintains a directed graph of currency conversion routes. Given available pairs (e.g. EUR/USD, USD/JPY), the system can compute indirect conversions (EUR/JPY) by traversing the graph — without requiring explicit direct pairs for every combination.

## Layer breakdown
**Backend**: `FxConversionRoute` table in `backend/app/db/models.py`. Graph traversal logic in `backend/app/services/fx.py`.
**Frontend**: `frontend/src/lib/stores/currencyGraphStore.ts` — client-side graph for UI use (e.g. showing conversion paths).

## Dependencies
- [[F-016]] FX Pair CRUD — pairs are the nodes/edges of the graph
- [[F-017]] FX Rate Sync — ensures edges have current rate data

## Enables
- [[F-024]] Assets — asset prices shown in user's preferred currency require conversion
- [[F-057]] Currency Conversion — the actual conversion calculations use this graph
- [[F-054]] Dashboard — portfolio total in single currency requires cross-pair conversion

## Implementation notes
- `FxConversionRoute` table stores: base, quote, provider assignments, priority ordering
- Graph traversal is depth-limited to avoid infinite loops on misconfigured cycles
- **Triangulation example**: EUR→USD→JPY gives EUR/JPY rate without an explicit EUR/JPY pair
- The graph is rebuilt on pair changes — no persistent graph structure, recomputed from DB rows
