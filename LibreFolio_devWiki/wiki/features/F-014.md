# F-014 — Image Upload & Crop broker icon

## What it does
Allows users to upload and crop a custom icon for each broker. The image upload goes through the general-purpose file upload API, then a cropping modal lets users select a square region. The resulting cropped image URL is stored as `broker.icon_url` and displayed in the broker card and detail page header.

## Layer breakdown
**Backend**: `backend/app/api/v1/uploads.py` (image upload + preview endpoint with size hints), `backend/app/services/static_uploads.py` (image validation, save)

**Frontend**: broker creation/edit modal with image crop component, `frontend/src/lib/components/ui/` (image crop modal), `frontend/src/lib/components/` broker-specific components

## Dependencies
- [[F-009]] Broker CRUD — icon stored as `broker.icon_url`
- [[F-011]] File Management — uses the uploads infrastructure

## Enables
- (visual improvement only)

## Implementation notes
- Accepted MIME types for broker icons: `image/jpeg`, `image/png`, `image/webp`, `image/gif`
- `UploadSecurityError` raised for non-image MIME types
- Crop is client-side (canvas API); only the cropped blob is uploaded
- `icon_url` stores a relative path served by the backend static file route
