# F-064 — Data Separation prod/test

## What it does
Completely isolated data directories for production and test environments. The active environment is selected via the `LIBREFOLIO_TEST_MODE` environment variable. When test mode is enabled, all data operations (DB, uploads, broker reports, logs) use `backend/data/test/` instead of `backend/data/prod/`. Switching modes resets the SQLAlchemy engine singletons.

## Layer breakdown
**Backend**: `backend/app/config.py` (`is_test_mode()`, `set_test_mode()`, `get_data_dir()`, `DEFAULT_PROD_DATA_DIR`, `DEFAULT_TEST_DATA_DIR`), `backend/app/db/session.py` (engine singletons reset on mode switch)

## Implementation notes
- Production: `backend/data/prod/{sqlite/app.db, custom-uploads/, broker_reports/{uploaded,parsed,failed}/, logs/}`
- Test: `backend/data/test/` — identical structure, isolated SQLite
- `LIBREFOLIO_TEST_MODE=1` env var switches mode; `set_test_mode(True)` programmatic switch
- `_reset_engine_singletons()` called on switch — recreates DB connection with new path
- Test server runs on port `TEST_PORT` (default 8001); prod on `PORT` (default 8000)
- `./dev.py server --test` starts server in test mode
