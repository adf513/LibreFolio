# F-017 — FX Rate Sync

## What it does
Fetches historical and current FX rates from configured providers and persists them in the `fx_rate` table. Sync is triggered manually via the FX list page ("Sync All") or per-pair via the detail page. The system tries providers in priority order (multi-provider fallback), stores one record per day per pair, and reports sync results per pair with status (success/partial/failed) and coverage details.

## Layer breakdown
**Backend**: `backend/app/api/v1/fx.py` (POST `/fx/sync` bulk), `backend/app/services/fx.py` (`sync_pairs_bulk()`), `backend/app/db/models.py` (`FxRate`: `base`, `quote`, `date`, `rate`, `provider_code`)

**Frontend**: sync button in FX list + detail pages, sync result toast notifications

## Dependencies
- [[F-016]] FX Pair CRUD — pairs define what to sync
- [[F-015]] FX Provider Registry — providers called during sync

## Enables
- [[F-023]] FX Chart — chart data comes from synced rates
- [[F-057]] Currency Conversion — conversion uses synced `fx_rate` table

## Implementation notes
- Daily-point policy: one `FxRate` record per (base, quote, date) — upserted with `INSERT OR REPLACE`
- `sync_pairs_bulk()` returns `FXSyncBulkResponse` with per-pair `FXSyncPairResult` (status, date range, provider used)
- Cache: `_fx_fetch_cache` (TTL 5min) prevents rate-limiting on provider APIs
- MANUAL rates not fetched — entered manually via FX detail page rate editor
- `truncate_fx_rate()` in `decimal_utils.py` normalizes rate precision
