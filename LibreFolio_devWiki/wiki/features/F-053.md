# F-053 — Scheduler Settings UI

## What it does
Admin-only section in `GlobalSettingsTab.svelte` for configuring the market data scheduler. Shows the scheduler's current state ("last run", "next run"), provides toggles and inputs for all scheduler `GlobalSetting` keys, and reads the scheduler state via `GET /api/v1/admin/scheduler/state` for the "Last execution" hints displayed under each input.

## Layer breakdown
**Frontend**: `frontend/src/routes/(app)/settings/` — admin settings tab, planned extension to `GlobalSettingsTab.svelte`

## Dependencies
- [[F-052]] Market Data Scheduler — the backend daemon being configured
- [[F-006]] Global Settings — scheduler settings are `GlobalSetting` entries

## Enables
- (user-visible scheduler control — no downstream features)

## Implementation notes
- Settings to expose:
  - `scheduler_enabled` (bool toggle)
  - `scheduler_current_price_frequency_minutes` (number input, 1–1440)
  - `scheduler_history_sync_time` (time input HH:MM, server time)
  - `scheduler_history_sync_horizon_days` (number input, 1–365)
- "Last execution" hint below each input reads from scheduler state endpoint
- State endpoint: `GET /api/v1/admin/scheduler/state` (read-only, admin-gated)
- Changes applied within 60s (scheduler re-reads settings each tick)
