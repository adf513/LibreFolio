# F-079 — GDPR Broker Access Compliance

## What it does
Refactoring of superuser data visibility to comply with GDPR principles. Currently superusers may have broader-than-intended visibility into other users' broker data. This feature formalizes what an admin can see vs. what requires explicit sharing.

## Implementation notes
- Audit all broker access control points in `backend/app/api/v1/brokers.py`
- Refactor superuser data visibility — admin sees admin data, not all user data
- May require new access level or explicit "superuser inspect" action with audit log
