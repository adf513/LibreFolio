# F-050 — File Preview System

## What it does
Inline preview for uploaded files in the Files page and the BRIM import modal. Supports multiple preview types: **image** (thumbnail rendering with the existing `PreviewCache`), **text** (raw content), **table** (CSV parsed as data grid), **markdown** (rendered HTML), and **code** (syntax-highlighted). Allows users to inspect a broker report file before triggering BRIM parsing.

## Layer breakdown
**Backend**: `backend/app/api/v1/uploads.py` (`GET /uploads/{file_id}/preview` — already exists for images, extension needed for text/CSV/markdown)

**Frontend**: Planned — new preview modal component, integrated in Files page and `BrokerImportFilesModal`

## Dependencies
- [[F-011]] File Management — builds on existing upload infrastructure

## Enables
- (usability improvement for file review before BRIM import)

## Implementation notes
- Image preview: already implemented via `PreviewCache` (LRU, TTL 1h, 50MB default)
- Text/CSV/markdown: served as raw content, rendered client-side
- `PREVIEW_CACHE_MAX_MB` env var controls image cache size
- Phase 7 Part 4b — absorbed from standalone `plan-phase7b-filePreview.md`
- Autonomous relative to transaction model changes — can be implemented independently
