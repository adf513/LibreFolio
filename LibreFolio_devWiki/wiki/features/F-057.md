# F-057 ‚Äî Currency Conversion triangulation

## What it does
Multi-hop currency conversion through a graph of configured FX pairs. When a direct rate (e.g., GBP‚ÜíCHF) is not available, the system finds a path through intermediate currencies (e.g., GBP‚ÜíEUR‚ÜíCHF) using the rates in `fx_rate`. This enables portfolio aggregation across any combination of currencies as long as there is a connected path in the configured pairs.

## Layer breakdown
**Backend**: `backend/app/api/v1/fx.py` (`POST /fx/convert` ‚Äî single conversion, `POST /fx/convert/bulk`), `backend/app/services/fx.py` (`convert_bulk()`), `backend/app/db/models.py` (`FxConversionRoute`)

**Frontend**: `frontend/src/lib/stores/currencyGraphStore.ts` ‚Äî client-side graph for triangulation in UI calculations

## Dependencies
- [[F-016]] FX Pair CRUD ‚Äî the configured pairs form the conversion graph
- [[F-017]] FX Rate Sync ‚Äî rates must be synced to convert

## Enables
- [[F-054]] Dashboard KPI ‚Äî cross-currency NAV aggregation

## Implementation notes
- `FXConversionRequest` / `FXConversionResult` ‚Äî single conversion request/response
- `FXConversionRouteResult` ‚Äî multi-hop path result with intermediate steps
- `currencyGraphStore.ts` in frontend: builds client-side adjacency map from configured pairs for triangulation hints
- `FXRouteStep` schema: each hop in the conversion path with rate + direction (forward/inverse)
- `BackwardFillInfo` included in response when exact date rate missing (uses nearest prior date)
