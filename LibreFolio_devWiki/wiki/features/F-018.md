# F-018 — FX Multi-Provider Fallback

## What it does
Each FX pair can have multiple provider assignments with a numeric priority (lower = higher priority). When syncing rates, the system tries providers in priority order. If the highest-priority provider fails or returns incomplete data, it falls back to the next provider automatically. This allows, for example, ECB→FED→BOE→SNB chains for broad currency coverage.

## Layer breakdown
**Backend**: `backend/app/services/fx.py` (`sync_pairs_bulk()` iterates provider list by priority), `backend/app/db/models.py` (`FxConversionRoute` includes provider assignments with `priority` field)

## Dependencies
- [[F-015]] FX Provider Registry — the pool of providers to fall back through
- [[F-016]] FX Pair CRUD — priority ordering configured per pair

## Enables
- [[F-017]] FX Rate Sync — fallback chain used during sync

## Implementation notes
- Provider assignment list stored in `FxConversionRoute` as JSON or related table with `priority` column
- MANUAL provider has `priority=999` — only used if all real providers fail
- Fallback is transparent to the user — the sync result shows which provider was actually used (`provider_code` in `FxRate`)
- `FXSyncLegDetail` in sync response reports per-provider attempt results
