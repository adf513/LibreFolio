# F-065 — JWT Cookie Auth stateless

## What it does
Stateless JWT-based authentication delivered via HttpOnly cookies. The JWT payload contains only the `user_id`; the user record is fetched from the database on every request. This design works correctly with multiple uvicorn workers — no shared session store required. The cookie TTL is configurable via `GlobalSetting.session_ttl_hours`.

## Layer breakdown
**Backend**: `backend/app/services/auth_service.py` (`create_jwt_token()`, `decode_jwt_token()`, `hash_password()`, `verify_password()`), `backend/app/api/v1/auth.py` (`SESSION_COOKIE_NAME = "session"`, cookie config constants)

## Implementation notes
- Cookie: name `session`, HttpOnly=True, SameSite=lax, Secure=False (set to True in HTTPS production)
- JWT payload: only `user_id` (UUID string) — minimal surface area
- `decode_jwt_token()` returns `user_id` or `None` on expiry/invalid
- Password hashing: `bcrypt` via `passlib`
- `SESSION_COOKIE_MAX_AGE = 60 * 60 * 24` (24h default) — overridden by `session_ttl_hours` global setting
- Stateless: no Redis, no DB session table — scales horizontally
