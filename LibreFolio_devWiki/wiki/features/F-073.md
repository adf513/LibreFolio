# F-073 — Backup & Restore

## What it does
Two-tier backup system: **per-series snapshot exports** (implemented) and **full portfolio export/restore** (placeholder, planned).

Per-series exports allow downloading CSV or JSON of individual data series before destructive operations: asset price history (`/backup/asset/{id}/prices`), asset events (`/backup/asset/{id}/events`), FX rates (`/backup/fx/{base}/{quote}/rates`). These are the "download before destructive operation" safety valves (e.g., before currency change wipe, pair teardown).

## Layer breakdown
**Backend**: `backend/app/api/v1/backup.py` (streaming CSV/JSON endpoints), `backend/app/services/backup_service.py` (`stream_rows_as_csv()`, `stream_rows_as_json()`, `BackupScope`, column definitions)

**Frontend**: download buttons on asset/FX detail pages before destructive actions

## Implementation notes
- Streaming responses (`StreamingResponse`) — no in-memory accumulation for large datasets
- `BackupScope` enum: `PRICES`, `EVENTS`, `FX_RATES`
- Column sets: `PRICE_COLUMNS`, `EVENT_COLUMNS`, `FX_RATE_COLUMNS` defined in `backup_service.py`
- `slugify_for_filename()` helper for safe download filenames
- Full portfolio export/restore: `GET /backup/export` and `POST /backup/restore` exist but return HTTP 501
