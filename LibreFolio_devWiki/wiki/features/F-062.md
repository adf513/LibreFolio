# F-062 — Docker Single-Image Deploy

## What it does
LibreFolio deploys as a single Docker image. The Python/FastAPI backend serves the pre-built SvelteKit frontend as static files and the MkDocs documentation site. The image is built on the host after running `./dev.py front build` and `./dev.py mkdocs build`, then `docker build -t librefolio .`.

`docker-compose.yml` provides a one-command deployment (`docker compose up -d`) with a bind-mount for persistent data at `./LibreFolio-data`.

## Layer breakdown
**Infra**: `Dockerfile` (Python 3.13-slim base, installs Python deps, copies backend + pre-built frontend/docs), `docker-compose.yml` (single service, port mapping, data volume, env_file)

## Implementation notes
- Base image: `python:3.13-slim`; system deps: `gcc libffi-dev git` (git needed for justetf pip dep)
- Build sequence: `./dev.py docker build` (calls front build + mkdocs build + docker build)
- Frontend served as static files from `frontend/build/` via FastAPI `StaticFiles`
- Docs served from `mkdocs_src/site/`
- Data dir inside container: `/app/backend/data/prod-docker` (bind-mounted to `./LibreFolio-data`)
- Ports: 8000 (prod), 8001 (test server, for `docker exec --test`)
- Health check: `GET /api/v1/system/health` every 30s
- `PORTFOLIO_BASE_CURRENCY`, `LOG_LEVEL`, `PORT` configurable via `.env`
