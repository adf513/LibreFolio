# F-013 — BRIM Plugins 11 broker parsers

## What it does
Eleven concrete broker report parser plugins, each handling a specific broker's export format. Each plugin extends `BRIMProvider`, is auto-discovered via `@register_provider(BRIMProviderRegistry)`, and implements `parse_file()` returning `(transactions, warnings, extracted_assets)` plus a `detect()` method for auto-detection.

## Layer breakdown
**Backend**: `backend/app/services/brim_providers/` — one file per broker:
- `broker_ibkr.py` — Interactive Brokers (`.csv`)
- `broker_degiro.py` — Degiro (`.csv`)
- `broker_directa.py` — Directa (`.csv`, `.xlsx`)
- `broker_etoro.py` — eToro (`.xlsx`)
- `broker_coinbase.py` — Coinbase (`.csv`)
- `broker_revolut.py` — Revolut (`.csv`)
- `broker_trading212.py` — Trading 212 (`.csv`)
- `broker_schwab.py` — Charles Schwab (`.csv`)
- `broker_finpension.py` — Finpension (`.csv`)
- `broker_freetrade.py` — Freetrade (`.csv`)
- `broker_generic_csv.py` — Generic CSV fallback (`.csv`)

Sample reports in `backend/app/services/brim_providers/sample_reports/` for testing.

## Dependencies

## Implementation notes
- Fake asset ID flow: plugins output transactions with **negative integers** as `asset_id` during parsing
- `plugin_version` field on each plugin — cache invalidation if plugin logic changes
- `detected()` method scans file headers/structure to identify the broker without user selection
- Warnings list (non-fatal): missing prices, unknown asset types, skipped rows
- All plugins tested via `./dev.py test external brim-providers`
