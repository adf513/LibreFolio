# F-046 — Transaction Model & Bulk API

## What it does
The backend transaction data model and atomic bulk API for recording investment transactions (BUY, SELL, DIVIDEND, INTEREST, FEE, TAX, TRANSFER_IN, TRANSFER_OUT, SPLIT, OTHER). Each transaction belongs to a broker and references an asset. The API enforces broker-level access control (EDITOR role required to create) and supports dry-run validation before commit.

## Layer breakdown
**Backend**: `backend/app/db/models.py` (`Transaction` model: `asset_id`, `broker_id`, `type`, `date`, `quantity`, `price`, `fees`, `currency`, `related_transaction_id` FK for bidirectional links), `backend/app/api/v1/transactions.py`, `backend/app/services/transaction_service.py` (`create_bulk`)

## Implementation notes
- Phase 7 Part 1 added `related_asset_event_id` FK between `Transaction` and `AssetEvent`
- Bulk API: `POST /brokers/{id}/transactions/bulk` (atomic per-broker), `validate` dry-run mode
- Access control: `GET/PATCH/DELETE /transactions` filter by user's `BrokerUserAccess` entries
- `related_transaction_id` self-FK — used for paired transactions (e.g., sell → buy in transfers)
- TransactionType enum: BUY, SELL, DIVIDEND, INTEREST, FEE, TAX, TRANSFER_IN, TRANSFER_OUT, SPLIT, OTHER
