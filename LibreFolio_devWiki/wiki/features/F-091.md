# F-091 — Multi-Worker Cache Server

## What it does
A dedicated cache process with Unix socket IPC, backed by `theine`, to share cache state across multiple uvicorn workers. Currently the 5-layer cache is per-process (in-memory), which means N workers = N independent caches with redundant provider calls.

## Implementation notes
- Dedicated process with Unix socket
- theine-backed (already used for single-process caches)
- Only needed for N>1 workers or PostgreSQL deployments
- LOW priority for current single-instance self-hosted use case
